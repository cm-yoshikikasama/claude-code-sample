# Markdown編集ルール

プロジェクト内のすべてのMarkdownファイルに適用

## 基本ルール

- Markdown太字記法（`**text**`）を使用しない
- 文章末尾にコロン（`文章:`）を使用しない
- 絵文字を使用しない
- URLを`<>`で囲まない（コード表記または通常のリンク形式を使用）
- 簡潔で明確な日本語を使用

## markdownlintルール

- MD040: コードブロックに言語指定（`python`, `typescript`, `text`等）
- MD036: 太字を見出しとして使用しない（セクション分けは`###`使用）
- MD022: 見出しの前後に空行
- MD025: 複数のH1見出しを使用しない
- MD032: リストの前後に空行
